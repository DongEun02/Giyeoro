# 기여로

기여로는 오픈소스에 처음 참여하는 사람이 자신에게 맞는 프로젝트와 이슈를 찾고, 실제 기여를 시작할 수 있도록 돕는 서비스입니다.

GitHub에는 수많은 프로젝트와 이슈가 있지만 처음 기여하는 사람은 어떤 저장소가 계속 관리되고 있는지, 어떤 작업부터 시작해야 하는지 판단하기 어렵습니다. 기여로는 사용자의 경험과 저장소의 최신 상태를 함께 분석해 선택지를 줄이고, 기여 과정을 단계적으로 안내합니다.

## 기여로가 해결하려는 문제

- 어떤 오픈소스 프로젝트를 선택해야 할지 판단하기 어렵습니다.
- 이슈가 많아 자신의 경험과 작업 범위에 맞는 일을 찾기 어렵습니다.
- 저장소가 현재도 관리되는지, 외부 기여를 잘 받아주는지 직접 확인해야 합니다.
- 이슈를 찾은 뒤에도 기여 절차와 코드베이스를 이해하는 데 시간이 필요합니다.

## 주요 기능

- **맞춤 라이브러리 추천**: GitHub에서 자주 사용한 언어, 최근 공개 활동, 참여 프로젝트의 공개 의존성을 분석해 경험과 맞는 오픈소스 라이브러리를 추천합니다.
- **단계별 첫 기여 탐색**: 작업 단계와 기여 언어를 먼저 선택한 뒤, DeepWiki에 등록된 오픈소스 프로젝트에서 문서·테스트·타입·버그·기능 이슈를 추천합니다.
- **기여하기 좋은 이슈 선별**: 담당자·연관 Pull Request·작업 의사 댓글이 없는 이슈를 확인하고, 난이도와 작업 유형을 함께 보여줍니다.
- **저장소 상태 확인**: 최근 개발 활동, 기여 가이드, 외부 Pull Request에 대한 관리자 응답을 바탕으로 저장소의 기여 친화도를 보여줍니다.
- **이슈 이해 지원**: GitHub 이슈의 핵심 내용, 예상 작업 범위와 주의점을 한국어로 정리합니다.
- **기여 준비와 진행 관리**: 저장소별 기여 가이드를 확인하고 관심·진행 중·완료 작업을 마이페이지에서 관리합니다.

## 추천 방식

기여로는 단순한 인기나 프로젝트 규모보다 사용자가 실제로 시작할 수 있는지를 우선합니다. DeepWiki에 등록된 프로젝트를 후보 카탈로그로 사용하되, 대형 프로젝트도 제외하지 않습니다. 대신 개별 이슈의 라벨·본문·논의량을 바탕으로 작업 범위와 난이도를 판단하고, 저장소가 최근에도 관리되는지와 기여 절차가 준비되어 있는지를 함께 확인합니다.

이슈 추천 전에는 지정된 담당자와 연결된 Pull Request뿐 아니라 최근 댓글의 작업 의사 표현도 확인합니다. 댓글 전체를 확인할 수 없거나 이미 다른 기여자가 작업을 맡겠다고 밝힌 이슈는 자동 추천에서 제외합니다.

추천 결과에는 판단 근거를 함께 표시해 사용자가 프로젝트를 직접 비교하고 선택할 수 있도록 합니다.

DeepWiki 후보 목록과 GitHub 메타데이터는 Neon Postgres의 저장소 카탈로그에 동기화합니다. 추천 요청 시 전체 목록을 다시 수집하지 않고 선택 언어의 후보만 조회하며, 실제 이슈 상태는 GitHub에서 최신 정보로 검증합니다.

## 데이터와 권한

맞춤 추천은 공개 GitHub 저장소와 공개 활동만 사용합니다. 비공개 저장소 권한은 요청하지 않습니다. 마이페이지는 GitHub의 기본 프로필 정보와 사용자가 저장한 작업 및 진행 상태를 계정별로 보관하며, 같은 계정으로 로그인하면 다른 브라우저나 기기에서도 이어서 관리할 수 있습니다.
